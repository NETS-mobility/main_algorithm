[백엔드측이 처리해야 할 것]
1. 예약정보 가져오기 (서비스 종류, 편도 여부, 날짜, 픽업시간, 병원도착시간, 출발시간)
2. 이용 가능한 차량 목록

[정리]
1. 편도: 자택->병원 == 병원 도착시간만
2. 편도: 병원->자택 == 병원 출발시간만
3. 왕복: 1시간 이하 == 배차 1번만 -> 문제 안 됨
4. 왕복: 1시간 이상 == 배차를 2번해야 되는데... twoWay.json으로 판별하자.....웅그래

[완료]
1. 예약자가 새로운 예약 정보를 등록했을 때, 해당 예약정보 가져오기: 이동방식(편도/왕복), 동행날짜 및 시간, 서비스(무브, 휠체어, 휠체어플러스) 선택 -> reservation.json
2. 예약자의 서비스(무브, 휠체어, 휠체어플러스) 확인 후 추가시간 a (= 10~20) 판단
3. 이용 가능한 차량목록 L(최초) 가져오기 (폐차/수리중인 차량은 목록에서 제외됨)
4. 첫번째 조건 확인:
// (If 편도(자택->병원)의 경우 or 왕복 첫번째 호출된 경우)
// 예약자의 희망 도착시간을 바탕으로, 예약자의 자택에서부터 병원까지의 소요시간 b (= tmap+20+a) 계산
// (If 편도 (병원->자택)의 경우 or 왕복 두번째(6.2번 과정) 호출된 경우)
// 병원에서 귀가 출발할 시간에 병원에서부터 드롭장소까지의 소요시간 d (= tmap+20+a) 계산

// (If 편도(자택->병원)의 경우 or 왕복 첫번째 호출된 경우)
// 예약자의 픽업 장소에 도착해야 되는 시간=예약자의 희망 도착 시각 - b을 계산
// (If 편도 (병원->자택)의 경우 or 왕복 두번째(6.2번 과정) 호출된 경우)
// 예약자의 병원에서 귀가 출발할 시간 확인
// 이용 가능한 차량목록 L(최초) 중 위의 경우에 따른 해당 시간(해당 시간에 예약 일정 수행 중인 차량은 미리 제외)의 각 차량 위치에서부터 픽업 장소(5.1의 경우) or 병원 장소(5.2의 경우)까지의 소요시간 계산->1차적으로 가능한 차량인지 판단->여러 개의 차량으로 구성된 차량 목록 A(1차적) 얻어냄
// (If 왕복 and 1시간 이하 병원동행의 경우->단일 배차)

//  두번째 조건 확인: 예약자의 병원에서 귀가 출발할 시간에 병원에서부터 드롭장소까지의 소요시간 c (= tmap+20+a) 더하여 계산
// (If 왕복 and 1시간 이상 병원동행의 경우 -> 편도 2번 배차)
//  4~5번을 반복하여 귀가 배차 추가 진행
// (If 편도의 경우) 8번 과정으로 이동
// 5번 과정을 만족하는지 확인 후 얻어낸 차량 목록A(1차적)을 바탕으로
// (If 왕복 and 1시간 이하 병원동행의 경우)
// 해당 차량이 6.1번 과정에도 만족하는지 추가로 확인. 만족하지 않는다면 차량 목록 A(1차적)에서 다음 차량 확인하는 과정 반복->최종적으로 가능한 차량인지 판단->여러 개의 차량으로 구성된 차량 목록 B(2차적) 얻어냄
// 얻어낸 차량 목록 B(2차적) or 편도의 경우 차량 목록 A(1차적) 중 시간 상수 계산하고, 이를 고려하여 알고리즘(최소시간 vs 효율->=시간 상수 만족하는 있으면, 시간 상수 만족하는 차량 중에서 여유 시간 판단하여 효율적인 차량 배차, 시간 상수 만족하는 차량 없으면, 최소 시간 차량 배차) 판단 과정 거쳐 최적 차량 1개 선택

// (If 병원동행시간 1시간 이상일 경우)
// 고위드유 매니저를 추가로 요청한다.

// 최적 차량의 전체 서비스 예측시간에 정돈시간(= 5분) 더함
